## 1. 什么是神经网络

神经网络是一类受生物神经系统启发的计算模型，由大量**神经元（Neuron）** 通过**加权连接**组成，通过学习数据中的模式来完成预测任务。

```
生物神经元                         人工神经元
   树突 ──→ 接收信号               x₁, x₂, ... → 输入特征
   胞体 ──→ 整合信号               Σwᵢxᵢ + b → 加权求和
   轴突 ──→ 输出信号               f(·) → 激活函数输出
```

**核心思想**：通过堆叠多层非线性变换，从数据中自动学习层次化的特征表示。浅层学到简单模式（边缘、颜色），深层学到复杂模式（物体部件、语义概念）。

## 2. 感知机：神经网络的基石

### 2.1 单层感知机（[[../Machine Learning/监督学习/感知机（Perceptron）|Perceptron]], 1958）

最简单的神经网络，只有一个神经元：

$$
y = f\left(\sum_{i=1}^{n} w_i x_i + b\right)
$$

- $x_i$：输入特征
- $w_i$：权重
- $b$：偏置
- $f$：激活函数（通常为阶跃函数）

```python
def [[../Machine Learning/监督学习/[[../Machine Learning/监督学习/感知机（Perceptron）|感知机（Perceptron）]]|[[../Machine Learning/监督学习/感知机（Perceptron）|perceptron]]]](x, w, b):
    return 1 if np.dot(w, x) + b > 0 else 0
```

### 2.2 感知机的局限 —— XOR 问题

单层感知机只能解决**线性可分**问题。对于 XOR（异或）这种非线性可分问题，它无能为力：

```
  XOR 真值表        可视化
  x₁ x₂  y        x₂
  0  0   0        1  ○  ●      ● = y=1
  0  1   1           (0,1) (1,1)
  1  0   0        0  ●  ○      ○ = y=0
  1  1   1           (0,0) (1,0)
                     0     1   x₁

  无法用一条直线分开!
```

> 💡 **解决方案**：堆叠多层（MLP），引入非线性激活函数。这是神经网络革命的起点。

## 3. 激活函数

没有激活函数，多层网络等价于单层线性变换。激活函数引入**非线性**，让网络能学习复杂模式。

| 激活函数 | 公式 | 特点 | 适用场景 |
|----------|------|------|----------|
| **Sigmoid** | $\sigma(x) = \frac{1}{1+e^{-x}}$ | 输出 (0,1)，两端梯度趋近 0 | 二分类输出层、门控 |
| **Tanh** | $\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$ | 输出 (-1,1)，零中心 | RNN 内部 |
| **ReLU** | $\max(0, x)$ | 计算快，缓解梯度消失 | **隐藏层默认首选** |
| **Leaky ReLU** | $\max(0.01x, x)$ | 解决 ReLU "死亡神经元" | ReLU 的改进替代 |
| **GELU** | $x \cdot \Phi(x)$ | 光滑，现代 Transformer 标配 | GPT、BERT |
| **Softmax** | $\frac{e^{x_i}}{\sum_j e^{x_j}}$ | 输出概率分布，和为 1 | 多分类输出层 |

```python
import numpy as np
import matplotlib.pyplot as plt

def plot_activations():
    x = np.linspace(-5, 5, 200)
    activations = {
        'Sigmoid': 1 / (1 + np.exp(-x)),
        'Tanh': np.tanh(x),
        'ReLU': np.maximum(0, x),
        'Leaky ReLU': np.maximum(0.1 * x, x),
        'GELU': x * 0.5 * (1 + np.tanh(np.sqrt(2/np.pi) * (x + 0.044715 * x**3))),
    }
    fig, axes = plt.subplots(2, 3, figsize=(12, 7))
    for ax, (name, y) in zip(axes.flat, activations.items()):
        ax.plot(x, y); ax.set_title(name); ax.grid(True)
    plt.tight_layout(); plt.show()
```

### ReLU 为什么好？

```
问题：Sigmoid/Tanh 在 |x| 大时梯度 ≈ 0 → 深层网络梯度消失，学不动
ReLU：x > 0 时梯度恒为 1 → 梯度畅通无阻
代价：x < 0 时梯度为 0（神经元"死亡"），Leaky ReLU 用 0.01x 挽救
```

## 4. 多层感知机（MLP）

### 4.1 结构

```
输入层          隐藏层1         隐藏层2          输出层
  x₁ ────●                           ●
           \                         / \
  x₂ ──────●───●───●───●───●───●───●───●─── ŷ₁
           /                         \ /
  x₃ ────●                           ●──── ŷ₂
           \                         / \
  x₄ ────●                           ●

  每个箭头 = 一个权重 w
  每个节点 = 加权和 + 激活函数
```

- **输入层**：接收特征，不进行计算
- **隐藏层**：特征提取与组合，层数/宽度是超参数
- **输出层**：根据任务输出预测（回归=1 个值，分类=类别概率）

### 4.2 万能近似定理

> 一个具有**至少一个隐藏层**且使用**非线性激活函数**的 MLP，只要有足够多的隐藏神经元，可以**以任意精度逼近任意连续函数**。

这是 MLP 的理论基础，但它不保证能通过训练学到该函数（需要数据、优化等配合）。

## 5. 前向传播与反向传播

### 5.1 前向传播（Forward Propagation）

数据从输入流向输出，计算预测值：

```python
# 三层 MLP 的前向传播
def forward(x, W1, b1, W2, b2, W3, b3):
    z1 = x @ W1 + b1        # 线性变换
    a1 = relu(z1)           # 激活
    z2 = a1 @ W2 + b2
    a2 = relu(z2)
    z3 = a2 @ W3 + b3
    y_hat = softmax(z3)     # 输出概率
    return y_hat
```

### 5.2 反向传播（Backpropagation）—— 神经网络的核心

反向传播是用**链式法则**高效计算所有参数梯度的算法：

```
目标：计算 ∂L/∂W（每个权重的梯度）
方法：从输出层向输入层逐层链式求导

示例（简化）：
  L = loss(ŷ, y)                    ∂L/∂ŷ ← 损失对输出求导
  ŷ = f₃(W₃·a₂)                     ∂L/∂W₃ = ∂L/∂ŷ · ∂ŷ/∂W₃  ← 链式法则
  a₂ = f₂(W₂·a₁)                    ∂L/∂W₂ = ∂L/∂ŷ · ∂ŷ/∂a₂ · ∂a₂/∂W₂
  a₁ = f₁(W₁·x)                     ∂L/∂W₁ = ...
```

**为什么叫"反向"？** 梯度从输出层往输入层传，和正向的输入→输出方向相反。

### 5.3 手动推导一个简单例子

```python
# 两层网络，MSE 损失，手动推导

# 前向:
# z1 = W1 @ x          (线性层1)
# a1 = sigmoid(z1)     (激活)
# z2 = W2 @ a1         (线性层2)  
# y_hat = z2           (输出, 回归任务)
# L = (y_hat - y)^2    (MSE)

# 反向（链式法则）:
# dL/dy_hat  = 2 * (y_hat - y)
# dL/dW2     = dL/dy_hat * dy_hat/dW2 = dL/dy_hat * a1^T
# dL/da1     = W2^T @ dL/dy_hat
# dL/dz1     = dL/da1 * sigmoid'(z1)    # sigmoid' = sigmoid * (1-sigmoid)
# dL/dW1     = dL/dz1 * x^T

# 参数更新:
# W1 = W1 - lr * dL/dW1
# W2 = W2 - lr * dL/dW2
```

### 5.4 计算图视角

```python
# PyTorch 自动帮我们做了反向传播
import torch

x = torch.tensor([1.0, 2.0], requires_grad=False)
W = torch.tensor([[0.5, 0.3], [0.2, 0.8]], requires_grad=True)

z = x @ W              # 前向: 构建计算图
loss = (z - 1).pow(2).sum()  # 计算损失

loss.backward()        # 反向: 自动计算梯度
print(W.grad)          # ∂loss/∂W 已经算好了
```

## 6. 损失函数

损失函数衡量预测与真实的差距，决定了"网络在学习什么"。

### 回归任务

| 损失函数      | 公式                               | 特点                   |
| --------- | -------------------------------- | -------------------- |
| **MSE**   | $\frac{1}{n}\sum(y - \hat{y})^2$ | 对离群点敏感，惩罚大误差         |
| **MAE**   | $\frac{1}{n}\sum$                | $y - \hat{y}$ 对离群点鲁棒 |
| **Huber** | 混合 MSE 和 MAE                     | 兼具两者优点               |

### 分类任务

| 损失函数 | 公式 | 特点 |
|----------|------|------|
| **Cross Entropy** | $-\sum y_i \log(\hat{y}_i)$ | **分类任务标准选择** |
| **Binary CE** | $-[y\log(\hat{y}) + (1-y)\log(1-\hat{y})]$ | 二分类 |

```python
# PyTorch 常用损失函数
import torch.nn as nn

nn.MSELoss()              # 回归
nn.L1Loss()               # MAE
nn.BCELoss()              # 二分类交叉熵（输入需经 sigmoid）
nn.BCEWithLogitsLoss()    # 二分类 + sigmoid 合并（数值更稳定，推荐）
nn.CrossEntropyLoss()     # 多分类 + softmax 合并（推荐）
```

> ⚠️ 为什么 CrossEntropyLoss 自带 Softmax？  
> 分开算数值不稳定（log(0) → -∞），合并后在数学上化简为 log-sum-exp，数值稳定得多。

## 7. 优化算法

### 7.1 梯度下降（Gradient Descent）

$$
\theta_{t+1} = \theta_t - \eta \cdot \nabla_\theta L(\theta_t)
$$

| 变体 | 计算梯度用的数据 | 特点 |
|------|-----------------|------|
| **BGD**（批量梯度下降） | 全部数据 | 精确但慢，内存爆炸 |
| **SGD**（随机梯度下降） | 1 个样本 | 快但震荡剧烈 |
| **Mini-batch SGD** | n 个样本（如 32/64） | **实际使用的标准方案** |

### 7.2 动量（Momentum）

在 SGD 基础上加入"惯性"，积累历史梯度方向：

$$
v_t = \beta v_{t-1} + (1-\beta)\nabla L(\theta_t)
$$
$$
\theta_{t+1} = \theta_t - \eta \cdot v_t
$$

效果：加速收敛，减少震荡，更容易冲出局部最优。

### 7.3 自适应学习率算法

| 算法 | 核心思路 | 推荐场景 |
|------|----------|----------|
| **AdaGrad** | 累积梯度平方，自动衰减高频参数的学习率 | 稀疏特征 |
| **RMSprop** | 指数移动平均替代累加，解决 AdaGrad 学习率→0 | RNN 训练 |
| **Adam** | 动量 + RMSprop = **当下最常用** | **几乎所有任务的默认选择** |
| **AdamW** | Adam + 解耦权重衰减 | Transformer 训练标配 |

### Adam 详解

```python
# Adam 算法伪代码
m_t = β₁  * m_{t-1}  + (1-β₁) * g_t        # 一阶矩估计（动量）
v_t = β₂  * v_{t-1}  + (1-β₂) * g_t²       # 二阶矩估计（自适应学习率）

m̂_t = m_t / (1 - β₁ᵗ)                       # 偏差修正
v̂_t = v_t / (1 - β₂ᵗ)

θ_t = θ_{t-1} - η * m̂_t / (√v̂_t + ε)       # 参数更新
```

```python
# PyTorch 使用
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)  # 推荐
```

### 7.4 学习率调度（LR Scheduler）

| 策略 | 说明 |
|------|------|
| **StepLR** | 每隔 n 个 epoch 将 lr × γ |
| **CosineAnnealingLR** | 余弦曲线衰减（现代标配） |
| **ReduceLROnPlateau** | 验证 loss 不降时自动降低 lr |
| **Warmup + Decay** | 先线性升到峰值，再余弦衰减（Transformer 标配） |

```python
# Warmup + Cosine Decay（Transformer 常用）
scheduler = torch.optim.lr_scheduler.OneCycleLR(
    optimizer, max_lr=1e-3, total_steps=total_steps
)
```

## 8. 正则化技术

防止过拟合，提升泛化能力。

### 8.1 L1 / L2 正则化

在损失函数中加入对权重大小的惩罚：

| 类型     | 额外项                  | 效果                                 |                            |
| ------ | -------------------- | ---------------------------------- | -------------------------- |
| **L1** | $\lambda \sum$       | $w_i$                              | 产生**稀疏权重**（很多 w→0），天然做特征选择 |
| **L2** | $\lambda \sum w_i^2$ | 权重值变小但不为零，**最常用**（也叫 weight decay） |                            |

```python
# L2 正则化在 AdamW 中就是 weight_decay 参数
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)
```

### 8.2 Dropout

训练时**随机丢弃**一部分神经元（不激活），迫使网络不依赖任何单个神经元：

```
训练时:                          推理时:
  ●───●───✕───●───✕               ●───●───●───●───●  (全部激活)
   \ / \ /     \ /                  
    ●   ●       ●                   每个权重 × (1-p) 做缩放
```

```python
nn.Dropout(p=0.5)    # 50% 丢弃率，常用在 MLP
nn.Dropout(p=0.1)    # 10% 丢弃率，Transformer 中的标准
```

> 💡 **为什么有效？** 相当于每次训练一个不同的"子网络"，推理时集成所有子网络（类似 Bagging）。

### 8.3 Early Stopping（早停）

验证集 loss 不再下降时停止训练，防止继续训练导致过拟合：

```
          ┌───────────────── Train Loss
Loss      │           ┌───── Val Loss
          │         ╱
          │       ╱
          │     ╱  ← 最佳点（提前停止）
          │   ╱         ╲
          │ ╱              ╲______  ← 从这里开始过拟合
          └─────────────────────── Epochs
```

### 8.4 Batch Normalization（批归一化）

在每个 mini-batch 内对特征做标准化：

$$\hat{x} = \frac{x - \mu_{batch}}{\sqrt{\sigma^2_{batch} + \epsilon}}$$

$$y = \gamma \hat{x} + \beta \quad （可学习的缩放和平移）$$

**效果**：
- 允许更大的学习率，加速收敛
- 一定程度上自带正则化（batch 统计量有噪声）
- 减轻对权重初始化的敏感性

| 归一化类型 | 归一化维度 | 适用场景 |
|-----------|-----------|----------|
| **Batch Norm** | 同一通道，跨 batch | CNN、MLP（batch 够大时） |
| **Layer Norm** | 同一样本，跨特征 | Transformer、RNN（batch 大小无关） |
| **Instance Norm** | 同一样本同一通道 | 图像风格迁移 |
| **Group Norm** | 分组归一化 | 小 batch 的 CNN |

## 9. 权重初始化

好的初始化让信号在前向和反向传播中保持稳定，防止梯度消失/爆炸。

| 方法 | 分布 | 适用激活函数 |
|------|------|-------------|
| **Xavier / Glorot** | $U\left(-\sqrt{\frac{6}{n_{in}+n_{out}}}, \sqrt{\frac{6}{n_{in}+n_{out}}}\right)$ | Sigmoid, Tanh |
| **He / Kaiming** | $\mathcal{N}\left(0, \sqrt{\frac{2}{n_{in}}}\right)$ | **ReLU 及其变体（推荐）** |

```python
# PyTorch 默认就是 Kaiming 初始化（大多数情况够用）
nn.Linear(in_features, out_features)  # 自动使用 Kaiming Uniform

# 手动指定
nn.init.kaiming_normal_(layer.weight, mode='fan_in', nonlinearity='relu')
nn.init.zeros_(layer.bias)
```

> 💡 **经验法则**：用 ReLU → Kaiming 初始化；用 Tanh → Xavier 初始化；用 Transformer → 小方差高斯（如 std=0.02）。

## 11. 训练技巧与超参数调优

### 11.1 数据预处理

```python
# 标准化（最常用）
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)   # 用训练集的均值和方差！

# 对于图像：除以 255 缩放到 [0,1] 或使用 ImageNet 统计量
```

### 11.2 超参数推荐范围

| 超参数 | 起点 | 搜索范围 |
|--------|------|----------|
| 学习率 | 1e-3 (Adam) | 1e-4 ~ 1e-2 |
| Batch Size | 32 或 64 | 16 ~ 256 |
| 隐藏层数 | 2 ~ 3 | 1 ~ 10 |
| 隐藏层宽度 | 128 或 256 | 64 ~ 1024 |
| Dropout | 0.2 ~ 0.5 | 0.1 ~ 0.5 |
| Weight Decay | 1e-4 ~ 1e-2 | 1e-5 ~ 1e-1 |

### 11.3 训练流程检查清单

```
□ 数据标准化/归一化
□ 验证集划分（如 80/20 或交叉验证）
□ 先在小样本上过拟合测试（关闭正则化，确保能过拟合 → 模型没有 bug）
□ 观察 loss 曲线（训练/验证是否同步下降）
□ 梯度检查（反向传播是否有问题）
□ 学习率预热 + 余弦衰减
□ 早停（patience = 10~20 epochs）
□ 保存最佳模型（按 val_loss，不是 train_loss）
```

### 11.4 调试：Loss 不下降怎么办？

```
1. 降低学习率（如从 1e-3 → 1e-4）
2. 检查数据（标签是否有问题？预处理正确吗？）
3. 调整 batch size（有时太大/太小都不好）
4. 检查梯度是否消失/爆炸（打印梯度范数）
5. 换个初始化方法
6. 检查激活函数是否合适（隐藏层不要用 Sigmoid）
7. 先用简单模型验证（去掉正则化，减少层数）
```

## 12. 卷积神经网络（CNN）简介

CNN 是为**网格结构数据**（图像、视频）设计的特殊网络。

### 核心操作

| 操作 | 作用 | 类比 |
|------|------|------|
| **卷积层（Conv）** | 用滑动窗口提取局部特征 | 多个滤镜扫描图像 |
| **池化层（Pool）** | 降采样，减少尺寸 | 压缩分辨率，保留关键信息 |
| **全连接层（FC）** | 高层推理 | 综合特征做决策 |

```
输入图像 (28×28×1)
     ↓ Conv2D(32, 3×3) + ReLU
特征图 (26×26×32)
     ↓ MaxPool(2×2)
特征图 (13×13×32)
     ↓ Conv2D(64, 3×3) + ReLU
特征图 (11×11×64)
     ↓ MaxPool(2×2)
特征图 (5×5×64)
     ↓ Flatten + Dense(128) + Dense(10) + Softmax
     ↓
分类输出（10 类概率）
```

### 为什么 CNN 比 MLP 更适合图像？

```
MLP 处理 224×224×3 图像:
  参数量 = 224×224×3 × 第一层神经元数 ≈ 150M 起步 😱
  而且位置信息被丢弃

CNN 处理同样的图像:
  一个 3×3×3 卷积核只有 27 个参数 😎
  天然保留空间结构（平移不变性）
```

## 13. 循环神经网络（RNN）简介

RNN 是为**序列数据**（文本、时序）设计的网络，核心是**隐藏状态的循环连接**：

$$h_t = \tanh(W_h h_{t-1} + W_x x_t + b)$$

```
展开的 RNN:
  x₁ → [h₁] → [h₂] → [h₃] → ... → [hₙ]
         ↑       ↑       ↑
        x₁      x₂      x₃            (h 从前一步传递到后一步)
```

### LSTM（长短期记忆网络）

解决 RNN 的梯度消失问题，引入三个**门控**：

| 门 | 公式 | 作用 |
|----|------|------|
| **遗忘门** | $f_t = \sigma(W_f [h_{t-1}, x_t] + b_f)$ | 决定忘记哪些旧信息 |
| **输入门** | $i_t = \sigma(W_i [h_{t-1}, x_t] + b_i)$ | 决定存储哪些新信息 |
| **输出门** | $o_t = \sigma(W_o [h_{t-1}, x_t] + b_o)$ | 决定输出哪些信息 |

LSTM 通过**细胞状态 C_t** 让信息几乎无损地跨越很长的时间步。

### 从 RNN 到 Transformer

```
RNN:   x₁→x₂→x₃→... (串行，O(n)路径长度)
LSTM:  x₁→x₂→x₃→... (串行但有门控保护)
Transformer: 每个位置直接关注所有位置 (并行，O(1)路径长度) ← 现在的主流
```

## 14. 动手实现：从零搭建 MLP

### 14.1 纯 NumPy 版（理解本质）

```python
import numpy as np

class MLP:
    def __init__(self, layer_sizes, activation='relu'):
        """
        layer_sizes: [输入维度, 隐藏1维度, 隐藏2维度, ..., 输出维度]
        """
        self.weights = []
        self.biases = []
        for i in range(len(layer_sizes) - 1):
            # He 初始化（适用于 ReLU）
            scale = np.sqrt(2.0 / layer_sizes[i])
            self.weights.append(np.random.randn(layer_sizes[i], layer_sizes[i+1]) * scale)
            self.biases.append(np.zeros(layer_sizes[i+1]))

    def relu(self, x):
        return np.maximum(0, x)

    def relu_derivative(self, x):
        return (x > 0).astype(float)

    def softmax(self, x):
        exps = np.exp(x - np.max(x, axis=1, keepdims=True))  # 减 max 防溢出
        return exps / exps.sum(axis=1, keepdims=True)

    def forward(self, X):
        """前向传播，保存中间值供反向传播使用"""
        self.cache = {'a0': X}
        a = X
        for i, (W, b) in enumerate(zip(self.weights[:-1], self.biases[:-1])):
            z = a @ W + b
            a = self.relu(z)
            self.cache[f'z{i+1}'] = z
            self.cache[f'a{i+1}'] = a

        # 输出层（不加激活，或 softmax）
        z_last = a @ self.weights[-1] + self.biases[-1]
        self.cache[f'z{len(self.weights)}'] = z_last
        y_pred = self.softmax(z_last)
        self.cache['y_pred'] = y_pred
        return y_pred

    def compute_loss(self, y_pred, y_true):
        """交叉熵损失"""
        m = y_true.shape[0]
        return -np.sum(y_true * np.log(y_pred + 1e-8)) / m

    def backward(self, X, y_true, lr=0.01):
        """反向传播"""
        m = X.shape[0]
        L = len(self.weights)

        # 输出层梯度 (softmax + cross entropy 合并)
        y_pred = self.cache['y_pred']
        dz = (y_pred - y_true) / m  # 合并后梯度就是这么简洁！

        grads_W = [None] * L
        grads_b = [None] * L

        for i in range(L - 1, -1, -1):
            a_prev = self.cache[f'a{i}']
            grads_W[i] = a_prev.T @ dz
            grads_b[i] = np.sum(dz, axis=0)

            if i > 0:
                da = dz @ self.weights[i].T
                dz = da * self.relu_derivative(self.cache[f'z{i}'])

        # 更新参数
        for i in range(L):
            self.weights[i] -= lr * grads_W[i]
            self.biases[i] -= lr * grads_b[i]

    def fit(self, X, y, epochs=100, lr=0.01):
        for epoch in range(epochs):
            y_pred = self.forward(X)
            loss = self.compute_loss(y_pred, y)
            self.backward(X, y, lr)
            if epoch % 20 == 0:
                acc = (np.argmax(y_pred, 1) == np.argmax(y, 1)).mean()
                print(f'Epoch {epoch}: Loss={loss:.4f}, Acc={acc:.4f}')


# 使用示例：MNIST 二分类简化版
if __name__ == '__main__':
```

### 14.2 PyTorch 版（工业级写法）

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

class MLP(nn.Module):
    def __init__(self, input_dim, hidden_dims, output_dim, dropout=0.3):
        super().__init__()
        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.BatchNorm1d(h_dim),      # 批归一化
                nn.ReLU(),
                nn.Dropout(dropout),
            ])
            prev_dim = h_dim
        layers.append(nn.Linear(prev_dim, output_dim))
        self.net = nn.Sequential(*layers)

    def forward(self, x):
        return self.net(x)


def train(model, train_loader, val_loader, epochs=100, lr=1e-3):
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model = model.to(device)
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.AdamW(model.parameters(), lr=lr, weight_decay=0.01)
    scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=epochs)

    best_val_loss = float('inf')
    patience_counter = 0

    for epoch in range(epochs):
        # Training
        model.train()
        train_loss = 0
        for X_batch, y_batch in train_loader:
            X_batch, y_batch = X_batch.to(device), y_batch.to(device)
            optimizer.zero_grad()
            loss = criterion(model(X_batch), y_batch)
            loss.backward()
            optimizer.step()
            train_loss += loss.item()

        # Validation
        model.eval()
        val_loss, correct, total = 0, 0, 0
        with torch.no_grad():
            for X_batch, y_batch in val_loader:
                X_batch, y_batch = X_batch.to(device), y_batch.to(device)
                output = model(X_batch)
                val_loss += criterion(output, y_batch).item()
                correct += (output.argmax(1) == y_batch).sum().item()
                total += y_batch.size(0)

        scheduler.step()
        val_loss /= len(val_loader)
        acc = correct / total

        if (epoch + 1) % 10 == 0:
            print(f'Epoch {epoch+1}: Train Loss={train_loss/len(train_loader):.4f}, '
                  f'Val Loss={val_loss:.4f}, Val Acc={acc:.4f}')

        # Early Stopping
        if val_loss < best_val_loss:
            best_val_loss = val_loss
            patience_counter = 0
            torch.save(model.state_dict(), 'best_model.pt')
        else:
            patience_counter += 1
            if patience_counter >= 15:
                print(f'Early stopping at epoch {epoch+1}')
                break

    model.load_state_dict(torch.load('best_model.pt'))
    return model
```
